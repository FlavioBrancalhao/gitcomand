Comandos do Git

Criar repositório

Comandos do Git para criar um repositório no GitHub e utilizar em um editor de texto como Visual Studio.

Comandos

Esses comandos são demonstrados apos criar um repositório no GitHub

  • git init - (Serve para iniciar o git em seu editor)
  • echo "Conteudo" >> index.html- (Serve para adicionar um arquivo em seu editor)
  • git add .- (Serve para adicionar git para ser commitado obs:tem um ponto ao final do add)
  • git commit -m "first commit"- (Serve para commitar primeira vez apos podera utilizar git commit -a -m "um texto" )
  • git branch -M main- (Serve para adicionar Branch main do seu projeto obs: antigamente era chamado de master)
  • git remote add origin https://seulink https encontrado no github- (Serve para adicionar um repositório/ caminho)
  • git push -u origin main- (Serve para enviar seu projeto ao repositório pela primeira vez após podera utilizar git push )

Removendo link do repositório

  • git remote -v - (Serve para verificar qual origin esta atrelado)
  • git remote rm origin - (Serve para remover a origin do seu repositório)

Clone do repositório

  • git clone https://seu lik . - (Clona em sua pasta o repositorio main obs: não esquça do ponto ao final do link dando espaço ponto)

Criando Branch

Branchs exemplificando de uma forma geek são as ramificações do multiverso do seu projeto.

  • git branch - (Mostra a lista das branchs que tem em seu projeto)
  • git branch name - (Cria branch dentro do projeto o name é o nome que você da a branch)

Deletando Branch

  • git branch -d name - (Deleta uma branch dentro do projeto o name é a branch a ser deletada)
  • git branch --delete name- (Deleta uma branch dentro do projeto o name é a branch a ser deletada)

Trocando de Branch

  • git checkout -b name - (Cria e troca uma branch nova automático aonde ja pode codar na nova branch)
  • git checkout name- (Troca de branchs apanas digite em name a branch que quer)

Enviando uma branch ao remote

  • git push origin nome_da_branch - (Envia uma branch ao remoto para ser verificada e baixada)
  • git fetch- (Verifica quais branchs foram enviadas)
  • git checkout –track origin/name- (Baixa branch do nome "name" do remoto para isso deve escrever origin/ e nome )

Juntando branchs a main

  • git merge name - (Junta a branch que esta escrita a atual mesclando os códigos)

Tags

Tags exemplificando de uma forma Gamer são os "saves" das branchs do seu projeto, entre outras palavras as versões.

  • git tag- (Mostra lista de tags)
  • git tag -a v1.0 -m "primeira alteracao"- (Cria uma tag com um titulo e uma descriçao com -m "descriçaõ")
  • git checkout v1.0- (Realiza a troca entre as tags)

Os mais utilizados

Comandos de Dia a Dia seram mais utilizados enquanto você desenvolve seu site ou sistema

  • git add .- (Adiciona todos os arquivos para ser commitado)
  • git commit -a -m "mensagem" - (commita todos os arquivos com uma mensagem)
  • git push- (Envia ao repositório)
  • git pull- (Baixa do repositório a sua pasta)

    • Esses Comandos é a ordem que deve fazer para enviar ao repositório até o push ja o pull envia
